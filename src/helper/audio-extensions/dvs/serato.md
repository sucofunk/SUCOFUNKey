# Serato Timecode Explained

## The Basic Concept
Serato timecode vinyl has a **continuous 1000 Hz tone** recorded on it. This tone encodes both **position** (where the needle is) and **direction** (forward/backward) using two clever techniques.

---

## 1. Direction Detection (Quadrature/Stereo Phase)
The left and right channels contain the **same 1000 Hz sine wave**, but offset by **90°**:

- **Left**: `sin(t)`  
- **Right**: `cos(t)` (same wave, shifted 90°)

When the record plays **forward**, the left channel leads the right. When it plays **backward**, the right leads the left. By tracking which channel crosses zero first, you can determine direction.

If you plot L vs R on an oscilloscope in X-Y mode, you see a **circle** — spinning one way for forward, the other for reverse.

---

## 2. Position Encoding (Amplitude + LFSR)
Position is encoded using **amplitude modulation**:

- **Bit 1** = Full amplitude
- **Bit 0** = Reduced amplitude (~25% quieter, a "notch")

These bits form a **pseudo-random sequence** generated by a **Linear Feedback Shift Register (LFSR)**. The key property: every 20-bit window in the sequence is **unique** across the entire record (~712,000 positions for Side A).

So once you read 20 consecutive bits, you can look up the **exact position** on the record. This is why timecode can "lock on" even if you drop the needle in the middle.

---

## 3. Decoding Steps
1. **Sample** the stereo audio (44.1–48 kHz typical)
2. **Track zero crossings** on both channels with hysteresis
3. **Determine direction** from phase relationship
4. **Extract bits** by measuring amplitude at specific phase points
5. **Lookup position** in a pre-computed LFSR table (~1M entries)
6. **Calculate speed** from timing between zero crossings

## Teensy Implementation Considerations

### Hardware
- **Teensy 4.1** with 16MB PSRAM
- **SGTL5000** audio codec for line-in

### Challenges
- **Lookup table size**: 20-bit LFSR needs ~4MB — PSRAM is perfect for this
- **Precise timing**: Zero crossings happen every ~0.25ms at normal speed
- **Real-time processing**: Teensy Audio Library handles audio I/O efficiently

### Signal Processing Pipeline
1. **Input**: Stereo 16-bit signed audio from SGTL5000 ADC
2. **DC offset removal**: IIR highpass filter or exponential moving average
3. **Zero crossing detection**: Compare against tracked zero level with hysteresis
4. **Quadrature tracking**: Monitor both channels' zero crossings
5. **Bit extraction**: Sample amplitude at specific phase points
6. **LFSR validation**: Compare extracted bits against expected sequence
7. **LUT lookup**: Pre-compute all LFSR values → position mappings in PSRAM
